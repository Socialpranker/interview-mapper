# Выход: Возможности и приоритизация (Opportunity Tree)

**Когда:** цель — превратить инсайты в приоритизированные возможности/решения (вход в роадмап).
**Вход:** инсайты из `score_insights.py` (с частота×критичность и статусом).

## Структура (по Opportunity Solution Tree)
```
ЦЕЛЬ (outcome): [измеримая бизнес/продуктовая цель наверху]
├── Возможность 1 (боль/потребность из инсайта) [приоритет: score]
│    Основана на: X из N интервью · критичность · verified-цитаты
│    ├── Решение A — [идея] · усилие:низк/сред/выс
│    └── Решение B — …
│    Как проверить: [assumption test]
├── Возможность 2 …
```

## Приоритизация
- Ранг возможности = **частота × критичность** из `score_insights` (не голос самого частого).
- Отдельная дорожка **watchlist**: редкое-но-критичное и напряжения — не хоронить под частым.
- Каждая возможность заземлена на ≥k интервью с verified-цитатой; иначе «гипотеза, добрать данные».

## Правила
- Не прыгай в решения без возможности (боли). Решение без заземлённой боли = вкусовщина.
- Для спорных приоритетов (совет флагнул) — на человека, не решай сам.
- Указывай усилие/риск решения, но НЕ выдумывай оценки, которых нет в данных.
