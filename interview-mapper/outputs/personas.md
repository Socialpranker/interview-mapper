# Выход: Персоны

**Когда:** цель — сегменты/персоны из серии интервью. Нужно ≥5 интервью, иначе персона = стереотип.
**Вход:** наггеты (с role/segment) → кластеризация по сегменту, не по теме.

## Правило заземлённости
Персона — не вымысел, а агрегат. Каждый атрибут персоны опирается на ≥k разных интервью с verified-цитатой.
Один яркий респондент ≠ персона. Помечай `[слабо: n<k]`, если данных мало.

## Формат персоны
```
## Персона: [архетип, не имя] — [сегмент]
Основана на: N интервью (роли/сегмент)
Цели: [что хочет] — «цитата» (Интервью, Lxx)
Поведение: [что реально делает, факты прошлого] — «цитата» (Lxx)
Боли: [pains, ранжированы] — «цитата» (Lxx)
Барьеры: структурные vs операционные
Триггеры/мотиваторы: [что двигает]
Отличие от других персон: [чем не похож]
Уверенность: высокая/средняя/низкая
```

## Правила
- Атрибут без ≥k подтверждений → не в персону, а в «гипотезы».
- Не додумывай демографию, которой нет в данных.
- Персоны должны РАЗЛИЧАТЬСЯ (если две почти одинаковы — это одна).
- Напряжения между сегментами фиксируй отдельно (важнее усреднения).
