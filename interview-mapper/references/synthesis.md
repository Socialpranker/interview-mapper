# Синтез инсайтов из серии интервью (S5–S7)

Стадия «N картирований → инсайты». Главный принцип: **собирать снизу из атомов, а не сжимать сверху**.
Суммаризация суммаризаций = место, где галлюцинации компаундируются («сотрудники считают X» из воздуха).

## S5 — Наггеты (атомизация)
Из каждого картирования вытащи атомарные наблюдения. Наггет неделим и заземлён.

Схема (`nuggets.json`, список):
```json
{"id":"n1","interview":"Дарья","role":"операции","cell":"К9",
 "observation":"Узнаёт о решениях постфактум ~40%",
 "quote":"звонят мне ночью...","line":135,
 "verified":true, "severity":4, "valence":"-", "cluster":null}
```
- `verified` — прошла ли цитата через `verify_quotes.py` (используй ТОЛЬКО verified-наггеты как доказательство паттерна).
- `severity` 1–5 — критичность/влияние (ставит модель, обоснуй).
- `valence` `+`/`-`/`0` — знак (нужно для детекции напряжений).
- `role` — обязательно: инсайты скоупятся по роли.

## Рубрика severity / valence (модель ставит по этой шкале, не наугад)
Скрипт `score_insights.py` зависит от этих полей — мусор на входе даёт мусор в скоринге. Ставь обоснованно:
- **severity 1–5** — влияние на цель организации/пользователя: 5 = блокер/системный риск; 4 = крупная боль; 3 = заметное трение; 2 = неудобство; 1 = косметика.
- **valence** — знак отношения в наггете: `+` (работает/ценит/за), `-` (боль/против/сломано), `0` (нейтральный факт).
Валентность нужна для детекции напряжений: один и тот же кластер с `+` у одной роли и `-` у другой = НАПРЯЖЕНИЕ.

## S6 — Кластеризация (affinity, делает модель)
Сгруппируй наггеты по смысловым темам, проставь `cluster` каждому. Правила:
- **Считаем разные интервью, а не цитаты.** Один человек, повторивший мысль трижды, — не паттерн.
- Не сливай в один кластер противоположные позиции — если роли расходятся, это НАПРЯЖЕНИЕ (ценнее консенсуса), помечай валентностью.
- Кластер не «тема вообще», а конкретная закономерность.

## S6.5 — Учёт надёжности (скрипт, не глазами)
`python scripts/score_insights.py nuggets.json --k 3`
Считает по кластеру: distinct-интервью, триангуляцию (≥k разных интервью с verified-цитатой), частоту×критичность, напряжение, статус:
- **insight** — триангулирован (≥k источников) → можно называть паттерном.
- **watchlist** — редкое, но критичное (severity≥4) ИЛИ межролевое напряжение → не хоронить.
- **weak** — одиночный источник, низкая критичность → анекдот, не паттерн.
Если N интервью мало (<k) — честно: паттернов не будет, только watchlist. Не выдавай watchlist за инсайт.

## S7 — Инсайт-карточки (делает модель из прошедших кластеров)
Только для `insight` и `watchlist`. Обязателен проход контр-доказательств.

Формат карточки:
```
### [ID] Заголовок инсайта — одно предложение
Формулировка: [наблюдение + интерпретация + следствие — не пересказ]
Держится для: [роли/сегменты] · НЕ для: [где не работает]
Распространённость: X из N интервью (роли: …) · Критичность: высокая/средняя/низкая
Тип: КОНСЕНСУС | НАПРЯЖЕНИЕ ([роль A] vs [роль B] — в чём именно)
Доказательства (только verified):
  - [Интервью, роль] «дословная цитата» (Lxx)
  - [Интервью, роль] «…» (Lxx)
Контр-доказательства: [кто не подтвердил / что противоречит / где тишина]
Уверенность: высокая/средняя/низкая  (по триангуляции + заземлённости)
Следствие / рекомендация: [что делать; для кого]
```

Правила честности карточки:
- Нет verified-цитат ≥k разных интервью → это НЕ инсайт, помечай «watchlist / нужно больше интервью».
- Каждая карточка проходит контр-доказательства: активно ищи, кто НЕ подтвердил и кто противоречит.
- «Тишина» (тему подняли не все, кого спрашивали) — тоже сигнал, фиксируй.
- Инсайт всегда скоуплен: «для фондовиков», а не «для сотрудников».

## S7.5 — Стабильность синтеза (опционально, как S3)
Прогони S6–S7 дважды в свежем контексте. Инсайты, которые не воспроизвелись (появились/исчезли между прогонами), помечай `[⚑ нестабильно]` — на человека. Кластеризация субъективна, как и eNPS.

## Ранжирование вывода
Сортируй: сначала `insight` по (частота×критичность), затем `watchlist` (в т.ч. напряжения и редкое-критичное), затем `weak` не показывай в основном отчёте (вынеси в приложение «сырые наггеты»).
