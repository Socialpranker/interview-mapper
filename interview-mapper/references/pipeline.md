# Пайплайн S1–S4 — подробно

## S1 — QA расшифровки
**Цель:** снять фактические ошибки расшифровки, не трогая аналитику. Эмпирика: вычитка чинит имена/системы/числа (Слой 1), а не тон/выводы.

Шаги:
1. `number_lines.py вход.(txt|docx)` → пронумерованный `*_nl.txt`.
2. Пройди транскрипт, выпиши кандидатов на искажение:
   - имена собственные (люди, отделы, системы, продукты, выставки),
   - числа (%, стаж, количество, цены),
   - «мусорные»/бессмысленные куски, где ASR явно сломался.
3. Для каждого кандидата — тег «какую ячейку задевает» (напр. система → К5/E4). Если ни одну — низкий приоритет.
4. Классифицируй правку:
   - **уверенная** (контекст однозначен: «КАИС/комисс» → КАМИС) — чини, занеси в лог `(было → стало, тип, обоснование)`;
   - **спорная** (без аудио не угадать: фамилия, спорное слово) — НЕ чини, вынеси в раздел «спорное, нужно аудио».
5. Сохрани: чистовик `*_вычитано_nl.txt` + `лог_вычитки.md`.

Тип правки помечай: [Ф] факт, [Р] диаризация (перепутан спикер).

## S2 — Интерпретация + заземлённость
1. Прочитай нужный `templates/<линза>.md`.
2. Заполни Слой 1 (1 прогон). Каждая ячейка: вывод + **дословная цитата** + `LNN`.
3. Собери `claims.json`: `[{"cell":"К5","claim":"кратко тезис","quote":"дословно","line":22}]`.
4. `verify_quotes.py --transcript *_вычитано_nl.txt --claims claims.json --out qcheck.json`.
   - `rejected` → цитата не в источнике: замени на дословную или убери тезис.
   - `line_mismatches` → поправь номер строки.
   - зафиксируй `verified_share` — понадобится как вес прогона в S3.
5. Entailment (сам как судья): для каждой пары тезис↔цитата — «цитата влечёт тезис?». Нет → ослабь/в «под вопросом».
6. Контрфактика + omission: «что противоречит выводам? какие фрагменты не покрыты?» → раздел «Пропуски/противоречия».

## S3 — Совет надёжности (Слой 2)
1. N изолированных прогонов Слоя 2 (по умолчанию 3), каждый — свежий субагент, транскрипт подаётся заново.
   Формат каждого: `{"А1":{"label":"НЕЙТРАЛ","text":"..."}, ...}`.
2. `consensus.py run1.json run2.json run3.json --weights w1,w2,w3` (веса = verified_share прогонов).
3. Результат:
   - `flagged` (ярлык разошёлся) → пометь ячейку `[⚑ спорно]`, вынеси на слепую адъюдикацию человека;
   - согласные → `[консенсус]`, бери консенсусный ярлык, текст возьми из наиболее заземлённого прогона.
