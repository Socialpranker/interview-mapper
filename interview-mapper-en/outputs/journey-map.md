# Output: Journey Map

**When:** the goal is the path of a visitor/employee/customer through stages, with emotions and friction.
**Input:** nuggets tagged by journey stage (not by theme).

## Stages (adapt to context)
- Visitor: Awareness → Arrival/entry → Exhibition → Add-on services → Exit → After (review/return).
- Employee/process: Initiative → Approval → Execution → Handover → Post-analysis.
- Customer: Awareness → Search → Choice → Onboarding → Usage → Retention.

## Format (stage × dimension matrix)
```
## Stage: [name]
Actions: [what they do]
Emotion: [+/−, intensity] — emotion line across stages
Pains/friction: [list] — «quote» (Interview, Lxx)
Delight points: — «quote» (Lxx)
Who's affected / transparency: [for an employee — do they find out in time]
Prevalence: X of N interviews
```
At the end — an **emotion line** (peak/dip across stages) and the **top-3 friction points** by frequency×criticality.

## Rules
- Every friction/delight point is a verified quote + interview.
- Different roles/segments may have DIFFERENT maps — don't average into one if the paths diverge.
- A stage with no data — "not covered by interviews" (don't make it up).
